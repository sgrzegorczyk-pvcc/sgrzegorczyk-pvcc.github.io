Chapter 6: Programming Language
Term Definition
machine language Sequences of instructions that are encoded as numeric digits.
debugging Locating and correcting errors in a written program.
assembler A program which converts mnemonic expressions into machine language instructions.
assembly language A mnemonic system for representing programs.
translator A program which translates high-level primitives into machine-language programs.
compiler A translator program which compiles several machine instructions into short sequences to simulate the activity requested by a single high-level primitive.
interpreter A program similar to a translator, but one that executes the instructions as they are translated.
cross-platform software Software that is independent of an operating system's design as well as the machine's hardware design and is therefore executable throughout a network.
natural language Languages such as English, German, and Latin which evolved over time without formal grammatical analysis.
formal language Languages such as programming languages which are precisely defined by grammars.
machine independence Not reliant on the characteristics of a particular machine.
high-level programming language A programming language which includes many kinds of abstractions.
object-oriented programming (OOP) A programming process in which a software system is viewed as a collection of units, called objects, each of which is capable of performing the actions that are immediately related to the object itself as well as requesting actions of other objects.
data types:
  • integer
  • float
  • string
  • Integer data consists of whole numbers, probably stored using two's complement notation.
  • Float data might contain values other than whole numbers, probably stored in floating-point notation.
  • String data consists of a string of characters.
primitive data type A data type which is included as a primitive in a programming language.
data structure The conceptual shape or arrangement of data.
Python collection data structures:
  • list
  • tuple
  • set
  • dictionary
  • List items are ordered, changeable, and allow duplicate values.
  • Tuple items are ordered, unchangeable, and allow duplicate values.
  • Set items are unordered, unchangeable, and do not allow duplicate values.
  • Dictionary items are ordered, changeable, and do not allow duplicate values.
  • From: W3 Schools: Python Tutorial
function A set of instructions for performing a task that can be used as an abstract tool by other program units.
local variable A variable declared within a function which can be referenced only within that function.
global variable A variable whose scope is not restricted to a particular part of a program.
source program The original form of a translated program.
object program The translated form of a translated program.
