Chapter 3 Terminology Table
Term Definition
operating system The software that controls the overall operation of the computer including file management, the user interface, and the program execution environment.
job queue A storage organization in which jobs are ordered in first-in, first-out fashion.
FIFO "First-In, First-Out": Objects are removed from the queue in the order in which they arrived.
interactive processing When operating systems allow a program being executed to carry on a dialogue with the user through remote terminals.
time-sharing When operating systems provide service to multiple users at the same time.
multiprogramming When time is divided into intervals and then the execution of each job is restricted to only one interval at a time.
application software The programs for performing tasks particular to the machine's utilization.
system software The programs for performing tasks common to computer systems in general.
utility software Programs for performing activities that are fundamental to computer installations but not included in the operating system.
graphical user interface (GUI) A user interface in which objects to be manipulated, such as files and programs, are represented pictorially on the display as icons.
shell A user interface that communicates with users through textual messages using a keyboard and monitor screen.
kernel The internal part of an operating system.
device driver Software units that communicate with the controllers to carry out operations on the peripheral devices attached to the machine.
file manager A software unit whose job it is to coordinate the use of the machine's mass storage facilities.
memory manager A software unit with the task of coordinating the machine's use of main memory.
virtual memory The large "fictional" memory space created by paging.
boot strapping A procedure that transfers the operating system from mass storage into main memory when a computer is turned on.
ROM "Read-Only Memory": Memory whose contents can be read but not altered.
deadlock The condition in which two or more processes are blocked from progressing because each is waiting for a resource that is allocated to another.
spooling A technique of holding data for output at a later but more convenient time.
super user (or administrator) A person with highly privileged access to the operating system through identification as the administrator during the login procedure.
multi-factor authentication The requirement of more than one piece of evidence to gain access to an account.
